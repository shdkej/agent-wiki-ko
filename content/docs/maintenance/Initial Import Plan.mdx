# 초기 가져오기 계획

## 가져오기 전략
1. 원본 markdown 노트는 `source/shdkej-content/`에 보존한다.
2. 먼저 카테고리 페이지를 만든다.
3. 시간별 실행마다 한 카테고리 또는 소수의 노트 묶음을 선택한다.
4. 요약, 링크, 개념 추출, 종합 페이지를 개선한다.
5. 모든 페이지는 500줄 이하로 유지한다.

## 비교 전략
- 사용자가 노트를 직접 수정하거나 다시 쓰면 그 버전을 `human-reviews/`에 저장한다.
- 인간 수정본과 현재 에이전트 페이지를 비교한다.
- 에이전트 버전을 무조건 덮어쓰지 말고 선호 차이를 추출한다.

## 재생성 전략
드리프트가 심해지면 원본에서 위키를 다시 만들고, 이전 경로와 새 경로를 비교한다.
