# 확장 전에 구조를 본다

A recurring operating idea: before increasing scope, count, devices, or systems, first check whether the existing platform or context boundaries already solve the problem.

## Why it matters
This concept appears repeatedly in recent work around architecture, bot separation, and system design. It helps prevent overbuilding.
